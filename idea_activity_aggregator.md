# Идея: Единая Лента Активности и Автоматический Учет Opex/Capex

## 1. Концепция
Расширение текущей системы Jira Timesheet до полноценного агрегатора активности сотрудника. Система собирает цифровые следы из различных корпоративных инструментов для формирования объективной картины рабочего дня, автоматической разметки типов деятельности и расчета финансовых показателей (Opex/Capex).

## 2. Источники данных (Data Connectors)

### 🔹 Confluence (Knowledge Management)
* **Что забираем:** Правки страниц, создание новых документов, комментарии.
* **Привязка:** По названию пространства (Space) или тегам к конкретным проектам/задачам.
* **Тип активности:** "Проектирование", "Документирование", "Аналитика".

### 🔹 GitLab (Development Activity)
* **Что забираем:** Коммиты, Merge Requests (создание, ревью, правки), комментарии в MR.
* **Привязка:** По ID задачи в тексте коммита или связи репозитория с проектом.
* **Тип активности:** "Разработка", "Code Review", "Bug fixing".

### 🔹 Google Calendar (Meetings & Events)
* **Что забираем:** Встречи, их длительность, состав участников, описание.
* **Логика:** Исключение личных событий, фильтрация по ключевым словам (Daily, Sync, Refinement).
* **Тип активности:** "Коммуникации", "Планирование", "Обучение".

### 🔹 Jira Cloud (Core Tasks)
* **Что забираем:** Текущие Worklogs (как есть) + переходы задач по статусам (Activity Stream).

## 3. Лента активности (Unified Activity Feed)
Визуальный таймлайн дня сотрудника, где "схлопываются" данные из всех источников:
* **09:30 - 10:00:** [Calendar] Daily Sync (Project A)
* **10:00 - 12:00:** [GitLab] 5 commits in Repo B, 2 reviews (Project B)
* **12:00 - 13:00:** [Confluence] Editing "System Architecture" (Project B)
* **14:00 - 15:30:** [Jira] Worklogged 1.5h on TASK-123

## 4. Автоматизация Opex/Capex

### Алгоритм классификации:
1. **Capex (Инвестиции):** Новая функциональность, разработка архитектуры, создание новых модулей в GitLab/Confluence.
2. **Opex (Поддержка/Операционка):** Исправление багов (Bug), техподдержка, регулярные встречи (Daily), обучение, отпуска.

### Механика расчета:
* Система сопоставляет активность с "инвестиционными" и "операционными" признаками задач/проектов.
* Предлагает сотруднику (или менеджеру) подтвердить автоматическую разметку в конце недели.
* Итоговый отчет формирует процентное распределение затрат по FTE (Full-Time Equivalent) на основе реальной активности, а не только ручного списания часов.

## 5. Техническая реализация (Предложение)
1. **Background Workers:** Добавление асинхронных коннекторов (модуль `services/connectors/`).
2. **Storage:** Расширение схемы БД для хранения сырых событий (Activity Logs) перед агрегацией в Worklogs.
3. **ML/Heuristics:** Простой классификатор на основе ключевых слов и типов задач для первичной разметки Opex/Capex.
4. **UI:** Новый раздел "Activity Feed" в Streamlit для предпросмотра и подтверждения данных перед финальным импортом в Timesheet.

---
*Эта идея позволяет минимизировать "забывчивость" сотрудников при заполнении таймшитов и дает бизнесу точные данные для финансового аудита.*
